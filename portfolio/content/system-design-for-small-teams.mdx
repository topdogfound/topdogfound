# System Design for Small Teams

Small teams lose speed when they over-architect too early.

## Design order

1. Correctness
2. Operability
3. Simplicity
4. Scale later

## Practical baseline

- Modular monolith
- Postgres as source of truth
- Queue for async jobs
- Measured caching only

Scale by removing bottlenecks, not adding services blindly.
