# Setting Up a Local LLM with Laravel Boost MCP for Faster Development

AI-assisted development shouldn’t require sending your entire codebase to the cloud. With a **local LLM** and **Laravel Boost MCP**, you can get fast, context-aware assistance while keeping your code private and your feedback loop tight.

This guide walks through a practical setup that works well for day-to-day Laravel development.

## Why local LLMs for Laravel?

Local models shine when you want:

- **Privacy-first** code understanding
- Zero API costs
- Low-latency responses
- Full control over context (routes, models, migrations)

When paired with **Laravel Boost MCP**, the model understands your project structure instead of hallucinating generic Laravel advice.

## What is Laravel Boost MCP?

Laravel Boost MCP is an **MCP (Model Context Protocol) server** tailored for Laravel projects. It exposes structured project context such as:

- Routes, controllers, models
- Migrations and database structure
- Service classes and config
- File-level semantics

This allows LLMs to reason about *your actual codebase*, not just Laravel in theory.

## Prerequisites

Before starting, make sure you have:

- PHP 8.2+
- A Laravel project
- Node.js 18+
- Docker (recommended)
- A local LLM runtime (Ollama works great)

## Step 1: Run a local LLM

Using **Ollama** is the simplest approach:

```bash
ollama pull qwen2.5-coder:7b
ollama run qwen2.5-coder:7b
````

Good model choices for Laravel:

* `qwen2.5-coder:7b`
* `deepseek-coder:6.7b`
* `codellama:7b`

These models understand PHP, MVC patterns, and REST APIs well.

## Step 2: Install Laravel Boost MCP

Inside your Laravel project:

```bash
npm install -g @continuedev/cli
```

Create an MCP config:

```yaml
name: Laravel Boost
version: 1.0.0
schema: v1

mcpServers:
  laravel:
    command: php
    args: ["artisan", "boost:mcp"]
```

This starts an MCP server that exposes Laravel-aware context to your LLM.

## Step 3: Connect MCP to your editor

If you’re using **VS Code + Continue**:

* Open Continue settings
* Add your local model (Ollama)
* Attach the Laravel Boost MCP server

Now your AI assistant understands:

* Which model handles a route
* Where business logic lives
* How migrations relate to Eloquent models

## Practical workflows

Once connected, you can ask things like:

* “Refactor this controller into a service layer”
* “Add validation DTOs for this request”
* “Explain how this migration impacts existing queries”
* “Generate tests for this repository”

The responses are **project-aware**, not generic snippets.

## Performance tips

* Use smaller models (6–7B) for daily work
* Limit context to `/app`, `/routes`, `/database`
* Restart MCP after major refactors
* Avoid animating entire vendor directories

## Wrap-up

Running a **local LLM with Laravel Boost MCP** gives you the best of both worlds: AI-assisted development without cloud dependency. It feels less like autocomplete and more like pairing with a developer who actually understands your codebase.

Once you try it, going back to stateless prompts feels painfully slow.
